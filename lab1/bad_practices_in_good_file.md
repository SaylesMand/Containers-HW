## 2 плохие практики по использованию контейнера Dockerfile.good

1. Использование COPY --chown=airflow:airflow dags/ и COPY --chown=airflow:airflow plugins/ без явного контроля за копируемым содержимым.

.dockerignore игнорирует только указанные паттерны, но не защищает от случайного копирования:
- Крупных бинарных файлов
- Временных файлов разработки
- Конфигурационных файлов с секретами, не попавших в .dockerignore
- Ненужных файлов, увеличивающих размер образа
- Чувствительных данных, если они не соответствуют паттернам в .dockerignore


2. В Dockerfile отсутствует копирование файлов зависимостей (requirements.txt, pyproject.toml, setup.py) перед установкой Python-пакетов.

При любом изменении в dags/ или plugins/ будет пересобираться весь образ с нуля, включая слой с apt-get update и установкой vim/htop


+ Также стоит добавить .dockerignore исключение для тестовых данных, дампов БД
