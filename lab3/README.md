# Ответ на вопросы:

## 1. Вопрос: важен ли порядок выполнения этих манифестов? Почему?

Порядок имеет значение
ConfigMap должен быть создан до Deployment т.к. Deployment ссылается на ConfigMap через envFrom configMapRef.name: postgres-configmap.

Если ConfigMap не существует, Pod может не запуститься или не получить переменные окружения.
Для Service порядок не критичен, т.к. если подов ещё нет, Service просто ждёт их появления.
Deployment должен быть создан после ConfigMap т.к. зависит от ConfigMap.

## 2. Вопрос: что (и почему) произойдет, если отскейлить количество реплик postgres-deployment в 0, затем обратно в 1, после чего попробовать снова зайти на Nextcloud?

### Если отскейлить количество реплик postgres-deployment в 0:

Все поды PostgreSQL будут остановлены или удалены.
Nextcloud потеряет соединение с БД.
Вылезут ошибки и таймауты.

### Потом когда возвращаемся к 1:

Создается новый под PostgreSQL
Восстановится доступность базы данных.
Будет задержка из-за иницализации.
Если при этом были сделаны какие-то изменения или данные не были сохранены, они могут быть утеряны или не доступны сразу.
